  • Hal Jordan = Parallax

    This plugin was originally written for the landing page of my startup, Infinite.ly, so this demo uses many of the same graphical assets. My initial goal with writing HJ=P was to create a parallaxing system that uses z-indexes to judge the relative distance between objects, as this is already how web designers are layering their DOM elements on top of each other. It makes sense to leverage the CSS that's already there. Here's how to use HJ=P on your pages.

  • Preparing Your Layout

    In order to prepare your page for HJ=P, you need to design your graphics with parallax in mind. Primarily this means using a lot of transparent PNGs that can be positioned on top of a large background graphic, similar to how the planets and asteroids on this demo page are arranged on top of the space background. Unfortunately this means you'll need to come up with an IE-compatible way to manage your PNGs, the precise strategy for which is beyond the scope of this plugin.

    Each planet and asteroid group in this demo is on a separate div, positioned absolutely, and assigned a z-index. The graphics are background-images of those divs. .space { position:absolute; width:100%; height:2000px; z-index:3; background:#000 url(../images/bg.jpg) 50% -120px no-repeat; top: 0px; left:0px; } .tarsonis { position:absolute; z-index:18; width:350px; height:288px; background:transparent url(../images/new_planet2.png) 0 0 no-repeat; top:900px; left:5%;} .char { position:absolute; z-index:7; width:350px; height:212px; background:transparent url(../images/new_planet.png) 0 0 no-repeat; top:300px; left:90%; } .aiur { position:absolute; z-index:10; width:250px; height:191px; background:transparent url(../images/aiur.png) 0 0 no-repeat; top:10px; left:10%; } .asteroids { position:absolute; z-index:40; width:600px; height:190px; background:transparent url(../images/new_asteroids4.png) 0 0 no-repeat; top: 600px; left:60%; } .asteroids2 { position:absolute; z-index:50; width:332px; height:150px; background:transparent url(../images/new_asteroids5.png) 0 0 no-repeat; top:750px; left:20%; } .asteroids3 { position:absolute; z-index:60; width:350px; height:147px; background:transparent url(../images/new_asteroids6.png) 0 0 no-repeat; top:700px; left:85%; }

    It's important to note that in order for HJ=P to do its magic, it will first take your graphics and muck with their CSS, changing position to "fixed," background-attachment to "fixed" and stretching the width and height to "100%" respectively. It will adjust the background-position values so that the relative position of your PNG within the layout will be largely unchanged.

  • Add the script to your page, somewhere, wherever.

    The only requirement is jQuery itself, but you knew that already, didn't you? Download the plugin from its github repository here. <script language="Javascript" src="/javascripts/jquery.js"></script> <script language="Javascript" src="/javascripts/jquery.hal-jordan-is-parallax.js"></script>

    Then it's just a matter of attaching $.parallax to your main container (most likely $('body')) inside a document.ready() block, thusly: $(function(){ $('body').parallax({ targets : ".parallax_target" }); });

    As you can probably guess, the targets parameter is a jQuery selector that enables the parallax effect for every element it matches. Any valid jQuery selector should work here.
    HJ=P works by looking at the individual z-indexes of all the target elements, and slowing down their scrolling speed based on this. Specifically: the lower a z-index is, the "further away" it is, therefore, the slower it should move. A target with a z-index of 2 will be 10x slower than a target with a z-index of 20.

    You can see the effect on this page's background, with the planets, asteroids and space background all moving at different speeds as the window is scrolled up and down. In this scenario, the furthest object is obviously the space background, with the asteroids being the closest. (Well, technically, the content you're reading is the closest, since that's on the highest z-index, but the parallax plugin doesn't touch that.)

  • Options

    Here's the parallax call again, but with all the options: $('body').parallax({ targets : ".parallax_target", orientation : "vertical", highest_z : 99, lowest_z : 1, force_height : 3500, force_width : 1000 });

    targets, possible : any valid jQuery selector, e.g., $(".content, .article")
    Just because you can use any valid jQuery selector, doesn't mean you necessarily have to be cute here. It's usually better to simply add a single class to all of the elements you want to apply the plugin to, and just select them that way.
    orientation, possible : "vertical", "horizontal"
    highest_z, possible : any integer
    lowest_z, possible : any integer
    force_height, possible : any integer, greater than your current container height
    force_width, possible : any integer, greater than your current container width
